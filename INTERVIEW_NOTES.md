# 面试说明

## 题目理解

题目要求不是简单生成一张图片，而是要面向常规 2D 游戏创作流程：策划或独立开发者需要快速得到可用素材，素材应当风格统一、轮廓清晰、透明背景、可直接导入 Unity / Godot / Cocos。

因此 Looty 的 MVP 重点放在四件事：

- 高效生成：输入一句话或少量参数即可生成。
- 低成本访问：纯前端部署，面试官打开 GitHub Pages 就能试。
- 风格一致：通过统一风格选择和一致性种子控制输出稳定性。
- 工作流适配：导出透明 PNG，而不是带背景的普通图片。

## 当前实现

当前版本是静态前端 MVP，没有接真实生图 API。原因是公开面试项目如果依赖 API Key、后端服务器或付费额度，面试官很可能无法稳定访问。

为了保证可评审性，项目使用 Canvas 实现了一个可运行的素材生成器：

- 支持剑、斧头、锤子、法杖、弓、盾牌、药水、金币/徽章。
- 支持像素风和粗线矢量风。
- 支持火、冰、雷、毒、圣光等属性识别。
- 支持同种子下保持轮廓一致。
- 支持 3 个候选变体，模拟真实生成工具常见的多候选工作流。
- 支持装备进化，即在同一类型和轮廓上添加属性效果。
- 支持导出透明 PNG。

## 开源参考

- SpriteForge：浏览器端生成、预览、下载、GitHub Pages 展示方式。
- Aseprite / Piskel / LibreSprite：像素素材工具常见的透明背景、预览、变体选择、导出流程。

本项目没有直接复制第三方代码，核心 Canvas 绘制、需求识别、Prompt 封装和交互逻辑均为当前项目实现。

## 如果接真实 AI

实际商业版本可以把 Canvas 生成器替换为后端 AI 流程：

1. 前端收集文本、素材类型、风格、种子。
2. 后端将参数封装为 Prompt。
3. 调用文生图或图生图模型。
4. 使用 `rembg` 或背景移除 API 生成 Alpha PNG。
5. 返回最终 PNG 给前端展示和下载。

当前 UI、Prompt 封装、状态流、下载逻辑都可以复用。

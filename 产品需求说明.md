2D 游戏素材生成工具MVP PRD
一、 产品概述
- 产品名称：Looty
- Slogan: Looty, Make games easy.
- 产品定位：一个专为独立游戏开发者设计的“零后期”AI 2D 资产生成台。
- 核心价值：将传统的【生图 -> PS抠图 -> 导进引擎】工作流，压缩为【输入文字 -> 直接获得透明 PNG 游戏素材】的一键操作。
二、 目标用户与核心痛点
- 目标用户：没有专职美术的独立游戏开发者、游戏策划、Hackathon 参赛团队。
- 解决的痛点：
  1. 普通 AI 生图带背景，无法直接作为游戏 Sprite（精灵图）使用。
  2. 普通 AI 画风不可控，不符合 2D 游戏工业标准（需要清晰边缘、纯白背景）。
  3. 想要生成同一把武器的“升级版”（比如从铁斧头变成火焰斧头），传统 AI 很难保持轮廓一致。
三、 功能需求范围 (MVP Scope)
(注：由于开发时间仅为 2 天，严格执行优先级打分。P0 必须完成，P1 根据时间选做)
1. 前端表单与交互 (P0)
- 文本输入框：用户输入核心物品描述（如：“一把木制大锤”）。
- 画风选择器 (Dropdown)：限制用户只能选择预设好的两种画风：
  - 美漫粗线矢量风 (推荐用于UI/道具，前端写死展示即可，点击显示待开发)
  - 复古 2D 像素风 (推荐用于像素游戏，我们就做这一种)
- 一键生成按钮：点击后触发工作流。
- 结果展示区：展示最终生成的 去背景透明 PNG 图片，并提供一键下载按钮。
2. 后台 Prompt 自动封装引擎 
- 功能描述：用户不需要懂 AI 提示词，后台自动将用户的简短输入包装成“工业级 Prompt”。
- 公式：[用户输入] + [画风提示词] + [强制约束条件：白底、独立道具] (详见第六部分)。
3. AI 生图与自动去背闭环
- 调用大模型接口：将封装好的 Prompt 发给生图 API（即梦 1.5）。
- 自动去背景：接收到生图 API 返回的图片后，系统不要展示给用户，而是立刻在后台调用 rembg 库，剔除图片白底，转为透明 Alpha 通道的 PNG。
- 交付标准：用户拿到的图，必须是能够直接拖入 Unity 等游戏引擎的透明图片。
4. “装备进化”图生图功能
- 功能描述：在已经生成的透明图片下方，提供一个“升级(Upgrade)”输入框。用户输入“附加火焰特效”，系统将当前图片作为底图（Base Image），调用生图 API 的图生图 (Image-to-Image) 接口，生成轮廓一致但带有新属性的进阶装备。并再次自动去背。

五、 技术栈
- 前端：Streamlit (纯 Python)。
- 后台：Python
- AI 生图 API：即梦 1.5 API (或其他有预算且生成速度较快的 API)。
- 图像处理与抠图：
  - rembg (Python 库，用于极速精准去背景)
  - Pillow (Python 图像库，用于图片的本地保存和格式转换)
